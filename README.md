# Salary Management Tool

A minimal yet production-ready salary management platform for HR teams managing 10,000 employees.

## What This Solves
- Employee lifecycle management (add, view, update, delete) through a usable UI.
- Salary insights by country and by job title in country.
- High-volume seed data generation for realistic usage and performance testing.

## Core Features
- Employee CRUD with server-side pagination, filtering, and search.
- Tab-based workspace (Employee List, Employee Form, Salary Insights, Employee Snapshot).
- Employee attributes:
  - Full name
  - Email (unique)
  - Job title
  - Department
  - Country
  - Salary
  - Currency
  - Employment type
  - Status
  - Hire date
- Salary insights UI:
  - Min, max, average salary by country
  - Average salary for a job title in a country
  - Additional HR metrics:
    - Headcount
    - Median salary
    - P90 salary
    - Total payroll
    - Top job titles by headcount
    - Employment type distribution

## Tech Stack
- Frontend: Next.js 14 + React + TypeScript
- Backend: Rails 7 API (`backend/`)
- Database: SQLite (`sqlite3` in Rails + `better-sqlite3` for seed/test tooling)
- Validation: `zod`
- Testing: `vitest`

## Quickstart
1. Install dependencies:
   ```bash
   npm install
   ```
2. Seed 10,000 employees:
   ```bash
   npm run seed
   ```
3. Start Rails backend:
   ```bash
   cd backend
   bundle install
   bundle exec rails server -p 3001
   ```
4. Start Next.js app (new terminal):
   ```bash
   cd ..
   npm run dev
   ```
5. Run tests:
   ```bash
   npm test
   ```
6. Production build:
   ```bash
   npm run build
   npm run start
   ```

## Seed Data
- Seed script: `scripts/seed.js`
- Name inputs:
  - `data/first_names.txt`
  - `data/last_names.txt`
- Full names are generated by combining first + last name values.
- Seeding is deterministic and transaction-based for speed.

## API Endpoints
- `GET /api/employees`
- `POST /api/employees`
- `GET /api/employees/:id`
- `PUT /api/employees/:id`
- `DELETE /api/employees/:id`
- `GET /api/insights/country?country=...`
- `GET /api/insights/job-title?country=...&jobTitle=...`
- `GET /api/insights/options?country=...`

## URLs To Test

## Test Coverage Focus
- Validation rules and query parsing
- Employee repository CRUD/filter behavior
- Salary insight calculations (country and job title)
- Seed script determinism and 10,000 employee generation

## Deployment
### Docker
```bash
docker build -t salary-management .
docker run --rm -p 3000:3000 -v $(pwd)/sqlite:/app/sqlite salary-management
```

The container seeds data on startup and serves the app on port `3000`.
